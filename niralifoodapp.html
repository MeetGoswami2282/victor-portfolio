Project Overview

Global Garner Vendor Dashboard is a mobile application designed to help vendors manage orders, commissions, bills, wallet transactions, and account details efficiently. The goal of this project was to simplify complex vendor workflows and improve clarity across financial and operational tasks while maintaining a clean, scalable UI.

Target Users

The application is primarily designed for:

  • Vendors who need a simple and efficient way to track orders, manage bills, and monitor daily earnings without unnecessary complexity.
  • Operations teams responsible for monitoring order progress, handling approvals, and ensuring smooth and timely order fulfillment.
  • Finance users who require clear visibility into commissions, wallet balances, and transaction history for accurate financial tracking.

Problem Statement

The existing system faced several usability and clarity issues:

  • Key information was buried under dense layouts, making it difficult for users to quickly identify what mattered most.
  • Bills, commissions, and wallet balances were presented without clear structure, requiring extra effort to interpret financial details.
  • Primary actions such as Approve, Dispatch, and Withdraw were not clearly highlighted, leading to slower task completion.
  • Frequently performed actions required unnecessary steps, increasing time and effort for everyday workflows.
  • Users could not easily understand order progress or payment status, reducing confidence and clarity in the system.

UX Challenges

From a UX perspective, the key challenges were:

  • Financial data needed to be structured and prioritized so users could quickly understand key values without reading dense details.
  • Order and payment states needed clear, consistent indicators to help users understand progress and required actions at a glance.
  • Multiple modules required unified layouts and interaction patterns to ensure a seamless and predictable user experience.
  • The challenge was to present rich information while maintaining visual clarity through hierarchy and progressive disclosure.

UX Challenges

To address the identified problems, the following solutions were implemented:

  • Introduced card-based layouts to separate information clearly
  • Used consistent color coding for order and payment statuses (Pending, Approved, Dispatched, Rejected, Paid)
  • Standardized typography, spacing, and button styles across the app
  • Highlighted primary actions with clear visual emphasis
  • Improved navigation clarity between Orders, Bills, Wallet, and Account sections
