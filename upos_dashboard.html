Universal Point of Sales (UPOS) Dashboard

Company

Global Garner

Role

UI/UX Designer

Platform

Web Dashboard

Year

2024

Project Overview

The UPOS Dashboard serves as a powerful hub for managing business performance and operations. It offers real-time tracking of sales, customer activity, cashback cycles, and commission earnings. Users can monitor transactions, analyze growth metrics, and manage structured workflows efficiently.

Target Users

The application is primarily designed for:

  • Business Vendors managing products, services, inventory, commissions, and digital vouchers.
  • Finance-focused vendors tracking wallet balances, commission payouts, invoices, and GST details.
  • Operational teams handling bill verification, service approvals, bookings, and compliance workflows.

Problem Statement

The existing system faced several usability and clarity issues:

  • Information was spread across multiple modules without clear prioritization.
  • Financial data (commissions, wallet balances, invoices, GST breakdowns) lacked structured visibility.
  • Approval-based workflows (bill review, service bookings, product approval) were difficult to track.
  • Status indicators (Pending, Approved, Rejected, Paid) were not clearly distinguishable.
  • High data density increased cognitive load and reduced scanability.
  • Navigation across modules required unnecessary effort.

UX Challenges

From a UX perspective, the key challenges were:

  • Designing a scalable interface capable of handling complex financial and operational data.
  • Structuring high-density information (tables, invoices, wallet logs) for quick scanning.
  • Creating clear workflow visibility for approval-based systems.
  • Maintaining consistency across diverse modules (Products, Wallet, Bills, Vouchers, Services).
  • Balancing data-rich dashboards with a clean and modern visual hierarchy.

Project Goals

To address the identified problems, the following solutions were implemented:

  • Build a centralized web dashboard for complete vendor lifecycle management.
  • Improve clarity of financial data including commissions, GST, and wallet activity.
  • Simplify approval workflows for bills, services, and products.
  • Enhance data scanability using structured layouts and status indicators.
  • Reduce operational friction in daily vendor tasks.
  • Create a scalable design system to support future feature expansion.

The Solution

The solution was built around reducing complexity and creating a more intuitive vendor experience.

  • Introduced a modular sidebar navigation system for clear section separation and quick access.
  • Designed card-based KPI summaries for performance metrics and wallet balances.
  • Structured financial data using clean table layouts with clear column hierarchy.
  • Implemented color-coded status badges (Pending, Approved, Rejected, Paid, Active, Unused) for instant recognition.
  • Simplified approval workflows with prominent action buttons (Accept, Reject, Pay Now, Withdraw).
  • Created a unified design system with consistent typography, spacing, and component patterns.
