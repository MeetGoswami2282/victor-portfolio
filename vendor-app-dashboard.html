Project Overview

Global Garner Vendor Dashboard is a mobile application designed to help vendors manage orders, commissions, bills, wallet transactions, and account details efficiently. The goal of this project was to simplify complex vendor workflows and improve clarity across financial and operational tasks while maintaining a clean, scalable UI.

My Role

I was responsible for the complete design lifecycle of the product, including:

  • Understanding business and user requirements
  • Defining user flows and information architecture
  • Creating wireframes and high-fidelity UI designs
  • Designing reusable components and layout patterns
  • Ensuring consistency across screens and states
  • Preparing designs for developer handoff

Target Users

The application is primarily designed for:

  • Vendors managing daily orders, bills, and earnings
  • Operations teams reviewing order status and approvals
  • Finance-focused users tracking commissions and wallet balances

Problem Statement

The existing system faced several usability and clarity issues:

  • Screens were overloaded with information and lacked visual hierarchy
  • Financial data (bills, commissions, wallet balance) was hard to understand at a glance
  • Important actions like Approve, Dispatch, and Withdraw were not visually prominent
  • Users needed multiple steps to complete common tasks
  • Inconsistent spacing, typography, and components caused confusion
  • Order tracking and payment status lacked transparency

UX Challenges

From a UX perspective, the key challenges were:

  • Making complex financial information easy to scan
  • Reducing cognitive load for frequent vendor actions
  • Designing clear status visibility for orders and payments
  • Creating a consistent experience across different modules
  • Balancing detailed data with a clean, minimal interface

UX Challenges

To address the identified problems, the following solutions were implemented:

  • Introduced card-based layouts to separate information clearly
  • Used consistent color coding for order and payment statuses (Pending, Approved, Dispatched, Rejected, Paid)
  • Standardized typography, spacing, and button styles across the app
  • Highlighted primary actions with clear visual emphasis
  • Improved navigation clarity between Orders, Bills, Wallet, and Account sections
